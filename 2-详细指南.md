# 云端转山 - GitHub Pages 部署指南

## 🚀 为什么用 GitHub Pages？

- ✅ 在中国大陆访问速度快
- ✅ 完全免费，无限流量
- ✅ 自动 HTTPS
- ✅ 可以绑定自定义域名
- ✅ 比 Netlify 在国内更稳定

---

## 📦 部署步骤（10分钟完成）

### 方法一：直接上传（最简单）

#### 步骤 1：创建 GitHub 仓库

1. 登录 https://github.com
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - Repository name: `cloud-kora`
   - Description: `云端转山 - 2026`
   - 选择 **Public** （必须公开才能用 Pages）
   - ✅ 勾选 "Add a README file"
4. 点击 "Create repository"

#### 步骤 2：上传文件

1. 在仓库页面，点击 "Add file" → "Upload files"
2. 拖拽这两个文件到页面：
   - `index.html`
   - `success.html`
3. 在底部填写提交信息：`初始提交`
4. 点击 "Commit changes"

#### 步骤 3：启用 GitHub Pages

1. 在仓库页面，点击 "Settings"（设置）
2. 左侧菜单找到 "Pages"
3. 在 "Branch" 下拉菜单：
   - 选择 `main`
   - 文件夹选择 `/ (root)`
4. 点击 "Save"
5. 等待 1-2 分钟

#### 步骤 4：访问网站

你的网站地址是：
```
https://你的用户名.github.io/cloud-kora/
```

例如：如果你的 GitHub 用户名是 `zhongliu`，网址就是：
```
https://zhongliu.github.io/cloud-kora/
```

### 方法二：使用 Git（推荐给熟悉命令行的用户）

```bash
# 1. 创建本地文件夹
mkdir cloud-kora
cd cloud-kora

# 2. 初始化 Git
git init
git branch -M main

# 3. 添加文件（把 index.html 和 success.html 放到这个文件夹）
git add .
git commit -m "初始提交"

# 4. 关联远程仓库（替换成你的用户名）
git remote add origin https://github.com/你的用户名/cloud-kora.git

# 5. 推送
git push -u origin main

# 6. 然后去 GitHub 仓库设置中启用 Pages（参考方法一步骤3）
```

## ⚙️ 表单提交方案

### 问题：GitHub Pages 是静态托管，不支持表单处理

### 解决方案：使用免费表单服务

我推荐 **Formspree**（在国内可访问）：

#### 设置步骤：

1. **注册 Formspree**
   - 访问 https://formspree.io
   - 免费账号每月 50 次提交

2. **创建表单**
   - 点击 "New Form"
   - 填写表单名称：`Cloud Kora Waitlist`
   - 复制你的表单 endpoint（类似：`https://formspree.io/f/xyzabc123`）

3. **修改 index.html**
   
   找到表单部分（约第 344 行）：
   ```html
   <form name="waitlist" method="POST" data-netlify="true" ...>
   ```
   
   改为：
   ```html
   <form name="waitlist" method="POST" action="https://formspree.io/f/你的ID">
   ```

4. **修改成功跳转**
   
   在表单中添加隐藏字段：
   ```html
   <input type="hidden" name="_next" value="https://你的用户名.github.io/cloud-kora/success.html">
   ```

## 🌐 绑定自定义域名（可选）

如果你有自己的域名（如 `cloudkora.com`）：

1. 在域名服务商添加 CNAME 记录：
   ```
   www -> 你的用户名.github.io
   ```

2. 在 GitHub Pages 设置中：
   - 输入你的域名：`www.cloudkora.com`
   - 等待 DNS 验证
   - ✅ 勾选 "Enforce HTTPS"

## ❓ 常见问题

### Q1: 网站更新了但看不到变化？
强制刷新：`Ctrl + Shift + R`（或 `Cmd + Shift + R`）

### Q2: 404 错误？
- 确认 Pages 已启用
- 检查分支是否选择 `main`
- 等待 1-2 分钟部署完成

### Q3: 表单提交不成功？
- 确认已注册 Formspree
- 检查 action URL 是否正确
- 查看浏览器控制台错误信息

### Q4: GitHub Pages 在国内慢？
可以使用：
- **Cloudflare Pages**（国内访问好）
- **Vercel**（国内访问一般）
- **Gitee Pages**（纯国内，但需要实名认证）
